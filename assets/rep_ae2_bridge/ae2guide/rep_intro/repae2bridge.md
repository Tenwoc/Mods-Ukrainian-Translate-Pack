# Міст Ме-Replication

<Row gap="20">
<BlockImage id="rep_ae2_bridge:rep_ae2_bridge" scale="8"></BlockImage>
</Row>

RepAE2Bridge – це компонент, який з'єднує МЕ систему AE2 з мережею матеріалів Replication, забезпечуючи безперебійну інтеграцію між двома системами.

## Функції

Міст діє як перекладач між двома системами, надаючи доступ до даних матеріалів Replication у МЕ системі AE2 та навпаки.

Це дозволяє:

- Переглядати кількість матеріалів у мережі реплікації через термінали AE2
- Запитувати реплікації через систему автоматичного вироблення AE2
- Налаштовувати пріоритети вироблення для оптимального управління ресурсами

## Розміщення

Щоб розмістити міст, виконайте такі дії:

1. Розмістіть блок мосту у точці, яка може досягти обох мереж
2. Під'єднайте кабель AE2 до одного боку блоку
3. Під'єднайте кабель Replication до іншого боку блоку

## Налаштування пріоритетів

Міст включає систему пріоритетів, яка дозволяє контролювати порядок обробки запитів на вироблення. Це особливо корисно, коли у вас є кілька мостів або ви хочете визначити пріоритети для певних типів предметів.

### Доступ до графічного інтерфейсу пріоритетів

1. **Клацніть ПКМ** по мосту, щоб відкрити інтерфейс налаштування пріоритетів
2. Введіть значення пріоритету в числове поле
3. Значення може коливатися від -2 147 483 648 до +2 147 483 647
4. Значення з вищим пріоритетом обробляються першими
5. Значення з нижчим пріоритетом обробляються пізніше

### Рекомендації щодо пріоритетів

- **Додатні значення (1-1000)**: Високий пріоритет, обробляється раніше
- **Нуль (0)**: Усталений пріоритет, звичайний порядок обробки
- **Від'ємні значення (від -1 до -1000)**: Низький пріоритет, обробляється пізніше

### Варіанти використання

- **Виробничі мости**: Встановіть високий пріоритет (100-500) для мостів, які обробляють критично важливі предмети
- **Резервні мости**: Встановіть низький пріоритет (від -100 до -500) для мостів, які забезпечують резервну потужність для вироблення
- **Експериментальні предмети**: Встановіть дуже низький пріоритет (-1000+) для тестування або експериментального вироблення

## Оптимальне налаштування

- Використовуйте лише одну мережу реплікації на міст
- Усталено міст може одночасно обробляти ~20 реплікаторів
- Для підтримки більшої кількості реплікаторів використовуйте процесор вироблення з великою кількістю блоків спільної обробки вироблення
- Налаштуйте пріоритети на основі ваших виробничих потреб та доступності ресурсів
